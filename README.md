# Character Tool

A local-first tool for exploring fictional character interiority through visible reasoning.

## What it is

Character Tool is a roleplay instrument, not a companion app. You load a fictional character, set a scene with a genre and setting seeds, and exchange messages with them. The tool's defining feature is that the character's internal machinery is visible: after each exchange, a side panel shows what the character privately noticed, how they appraised the situation against their desires and fears, and what they decided to do about it. You watch the character think, not just speak.

It is built for studying how a character is moved by what happens and what they do about it.

## How it works

Each turn runs two model calls in sequence.

The first call, the appraisal pass, takes the user's message and works out what it means to the character: which desire or fear it activates, how threatening or welcome it is, the character's raw reaction and how the relationship reweights that reaction, what the character plans to do, and how hard they intend to push. The result feeds the inner-state panel.

The second call, the voice pass, receives the appraisal result and writes the in-character reply, conditioned on what the appraisal concluded.

Three features hold the character accountable across a scene. The scene objective gives the character a concrete goal targeting the other person, with a named obstacle: characters work toward something real rather than just reacting. The resistance governor keeps characters from changing too fast: a deep fear or lifelong pattern does not dissolve in a single exchange, and the appraisal prompt enforces this explicitly. The scene-fact ledger records what has been established as true and carries it forward so the character cannot forget or contradict it.

## Features

- Visible inner-state panel: the full appraisal output (perception, appraisal, reaction, tension, plan, chosen move) is displayed after each reply, so the character's reasoning is legible rather than hidden.
- Two-pass appraisal and voice: reasoning and speech are separate model calls, each with its own prompt and temperature.
- Scene setup with genre and randomized seeds: location, time of day, weather, mood, and situation are drawn from a seed list and can be rolled or chosen manually.
- Constrained scene objectives: objectives are generated by the model but must target the other person, name a real obstacle, and serve one of the character's core desires.
- Resistance governor: a pull-back step in the appraisal prompt enforces that characters do not open, soften, or change faster than a lifelong pattern allows.
- Scene-fact ledger: established facts are extracted after each exchange and injected into subsequent appraisals so the character cannot forget them.
- Branching and regeneration: any turn can be regenerated to produce an alternative reply, stored as a variant alongside the original, with navigation between variants.
- Multi-provider support: run locally via Ollama with no key required, or connect to OpenAI, Anthropic, Google Gemini, xAI Grok, or Groq.
- Persona system: characters are defined by a structured schema covering identity, desires, fears, standards, voice, and temperament, and can be authored by hand or imported from a free-text description.

## Quickstart

1. Install [Ollama](https://ollama.com) and pull a model.
2. Clone or download this repository.
3. Create a Python virtual environment and install the dependencies (see [SETUP.md](SETUP.md)).
4. Run `run.bat` from the repository root.
5. Open `http://127.0.0.1:8000` in your browser.

See [SETUP.md](SETUP.md) for the full installation path, including cloud provider setup.
